Brad Frost

The corpus's anchor voice on design-system fundamentals — from the "official story" definition through token architecture to agentic readiness.

Who he is in this corpus

Brad Frost appears across three sources spanning 2024–2025, making him the connective thinker linking foundations, tokens, and the agentic frontier. His throughline is separation of concerns: keep structure, behavior, and aesthetics independent so a system can flex across brands, platforms, and now machines.

Core positions

  • Definition. A design system is "the official story of how an organization designs and builds user interfaces," carried by a design library, a code library, and a reference website.1 See What Is a Design System?
  • Architecture. Web Components for cross-framework reach; a base/recipe "layer cake"; build on native standards for longevity.1 See Component Architecture.
  • Tokens. A three-tier model (primitives → semantic → component), themes as collections of decisions, tokens as the API for the design language, "flexibility, not uniformity."2 See Token Architecture.
  • Agentic. Design systems must become machine-readable infrastructure; success rests on coverage and validation; teams should build the "constraint system" and "quality pipelines."3 See Agentic Readiness.

Where he sits in the debates

  • vs. Dan Mall on consistency: Frost reframes the goal as flexibility/expression unlocked by structure, complementing Mall's "consistency is a by-product" stance.2 See Success Metrics.
  • vs. the type-safety camp: Frost bets on web standards for longevity; McMahon bets on the type system for correctness. See Type-Safe Design Systems.

Recommended reading he cites

Donella Meadows' Thinking in Systems, for interconnected hierarchies and feedback loops — see Systems Thinking.1
