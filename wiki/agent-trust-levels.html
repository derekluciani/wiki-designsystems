Governance & Trust Levels for Agents

Giving agents write access without guardrails is unacceptable risk — so teams define trust per action, scaled to the blast radius of the change.

Trust levels per action

A recurring lesson: granting agents direct write access to tokens and component APIs without guardrails creates unacceptable risk. Teams instead define trust levels per action, scaled to risk:1

Trust levelApplies toExamples
Auto-mergeLow-riskDoc typos, non-semantic refactors, simple a11y label fixes.
Draft PR onlyMedium-riskToken value changes, component copy updates, non-breaking prop additions.
Suggest / comment onlyHigh-riskNew components, API redesigns, cross-brand governance rules.

Hard architectural limits

Some systems make the boundary structural rather than advisory: GitHub's Primer explicitly constrains agents to creating issues or PRs but never merging, enforcing human review as a hard rule.1 The agentic-2026 session echoes this — keep agents in "suggest" or "draft PR" modes while humans retain the merge.2

Encoding governance as rules

Governance work is increasingly encoded as machine-readable rules agents must follow, letting design leaders guarantee certain decisions (e.g., "no new button styles") instead of relying on education.1 This is the agentic continuation of Dan Mall's WHO/WHAT/WHEN governance — see Governance: Conversations, Not Flowcharts.

Keep workflows in "suggest/draft" mode

The token/foundation use cases follow the same discipline: agents detect drift, propose semantic alignments, and simulate cross-brand theming, but stay in "suggest" or "draft PR" modes with final merges left to humans — a workable balance between scale and safety.1

Perspectives: autonomy vs. control

The defining tension of agentic design systems. The capability pull is toward autonomy (let agents ship); the governance pull is toward control (humans hold the merge). The corpus resolves it by tiering by blast radius rather than choosing globally: autonomy where a mistake is cheap and reversible (typos, labels), control where it's expensive and propagating (new components, cross-brand rules).1 For a commercial / multi-brand retail context, the report explicitly counsels starting on the conservative end — suggest and draft, not autonomously ship.1
