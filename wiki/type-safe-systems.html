Type-Safe Design Systems

Using the type system as a guardrail — forbidding invalid component usage at compile time instead of hoping documentation is read.

Typed tokens

Kathleen McMahon advocates a token pipeline (Figma → Style Dictionary) that generates TypeScript definitions, preventing silent failures and keeping multi-brand systems in sync.1 This is the type-system continuation of the token pipeline.

Discriminated unions and never

For component props, the recommendation is to use interface for common shapes and type for variants, and to deploy the never type to forbid invalid prop combinations. The type system thereby acts as a guardrail for accessibility and correct usage — invalid states become unrepresentable.1

Mall frames this as the move from documenting intent to encoding it: prose can say which prop combinations are invalid, but a component API that rejects those combinations at the type level turns guidance into an enforced system boundary.2

Type-safe styling

Approaches like tailwind-variants or CVA (Class Variance Authority) ensure styles only apply to valid variants, preventing CSS bloat and configuration errors.1

Runtime safety nets

Static types are not enough alone. McMahon adds an assert layer as a secondary defense: it catches issues via console errors in development that static types might miss.1 Belt and suspenders.

Perspectives: the as prop debate

McMahon takes a strong, quotable stance against the polymorphic as prop — calling it "the driftwood of component design" — because it blurs buttons and links and erodes semantic clarity. Her prescription: ship distinct button and link components rather than one shape-shifting component.1

This is a real tradeoff against the flexibility instinct elsewhere in the corpus. Frost's layer cake and Spotify's Encore tiers (see Team Maturity) both lean toward giving consumers more compositional flexibility; McMahon argues some flexibility (polymorphism) costs more in correctness than it returns. The reconciling principle: offer flexibility through typed, bounded APIs, not through escape hatches that defeat the types.
