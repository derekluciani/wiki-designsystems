Video title: Scalable scales that scale - Nate Baldwin at Into Design Systems
Date: 04/2025
Url: https://www.youtube.com/watch?v=v6sG0oElloc

In his presentation "Scalable scales that scale," Nate Baldwin outlines a systematic, parametric approach to building responsive and adaptive design systems. He argues against rigid pixel-based grids in favor of proportional, mathematical models. Here are his core recommendations:

1. Adopt a Systems-Oriented Design Method
Break down problems (Encapsulation): Instead of viewing a system as a whole, decompose it into smaller, manageable concerns (e.g., responsiveness, accessibility, component density). This separation allows you to solve each problem independently (16:53 - 17:13).
Use parameterization: Define system elements based on constants (like base sizes and ratios) rather than static, hard-coded values. This makes the system inherently scalable and dynamic (22:48 - 23:39).
Create abstractions (Archetypes): Rather than designing components individually, identify commonalities to create a core "archetype" or template. Use this as a foundation for building all other components (27:33 - 28:03, 33:30 - 33:50).
Illustrate with diagrams: Use system diagrams to clarify relationships between tokens, variables, and components. This is essential for communicating the structure of complex systems (34:26 - 34:55).

2. Implementation Strategies
Ditch pixel grids: Baldwin explicitly criticizes rigid pixel grids as non-proportional and prone to inconsistency. Instead, he suggests using modular scales (using a base value and a ratio) to generate values (13:42 - 14:14, 18:36 - 19:48).
Implement responsive typography: Use conditional ratios and base font sizes that adapt based on the viewport width (e.g., small, medium, large, widescreen). Use REM units for text to ensure the system respects the user's browser-level font size preferences for accessibility (20:10 - 21:55).
Proportional spacing: Use the same modular scale formula for spacing as you do for typography. This ensures that spacing remains proportional to text size, which is critical for maintaining visual relationships (25:19 - 25:52).

Differentiate Size vs. Density:
Size: Affects the overall scale of an element, including text.
Density: Affects only the internal padding of a component. By separating these dimensions, you ensure that content density changes do not break content hierarchy (28:06 - 29:18).
Define padding via relative indices: In your component archetypes, define padding using relative indices (e.g., -4, -1, 0, 1) rather than hard-coded pixel values. This makes padding responsive and easily adjustable via a scale (29:32 - 30:42).

3. Recommended Tools
Baldwin suggests exploring tools that support parametric design:
Proportio: An open-source tool he created to help designers build proportional typography, spacing, and component systems parametrically.
Utopia.fyi: Recommended for generating CSS clamp functions for fluid, proportional scaling across viewports.
Token Studio: Useful for building tangible, editable graph-based representations of your design system structure.
